# PRD: 문의하기(inquiry) 기능
- 작성일: 2026-04-19
- 완료일: 2026-04-19
- 상태: Done

## 배경 & 목적

사용자가 서비스 관련 문의를 접수할 수 있는 창구를 제공한다.
관리자는 접수된 문의를 확인하고 이메일로 답변을 발송하며, 처리 상태를 관리한다.

## 요구사항

### 기능 요구사항

**사용자 (공개 API)**
- [ ] 문의 제출 — 제목, 본문, 답변받을 이메일 3개 필드

**관리자 페이지 (Thymeleaf SSR)**
- [ ] 문의 목록 조회 (상태 필터: 전체 / PENDING / ANSWERED / SEND_FAILED / CLOSED)
- [ ] 문의 상세 보기 (제목, 본문, 이메일, 제출일시, 상태, 답변내용)
- [ ] 답변 작성 제출 → 답변 내용 DB 저장 + 이메일 발송
  - 발송 성공 → 상태 ANSWERED
  - 발송 실패 → 상태 SEND_FAILED (답변 내용은 저장됨)
- [ ] SEND_FAILED 상태에서 재발송 → 이메일 재발송
  - 재발송 성공 → 상태 ANSWERED
  - 재발송 실패 → 상태 SEND_FAILED 유지
- [ ] 닫기 → 이메일 미발송, 상태 CLOSED

### 비기능 요구사항
- 이메일 발송: spring-boot-starter-mail + Gmail SMTP (계정 정보는 환경변수)
- 답변 저장과 이메일 발송 분리 — 이메일 실패해도 답변 내용은 DB에 보존
- 비로그인 사용자도 문의 제출 가능 (인증 불필요)

## 가정 & 미확정 사항
- 가정: 관리자 1명이 처리 (담당자 배정 불필요)
- 가정: 문의당 1회 답변, 재발송은 같은 답변 내용으로만

## 범위 외
- 문의 삭제 (CLOSED 상태로 충분)
- 첨부파일
- 알림 (Slack, 푸시 등)
- 답변 내용 수정 후 재발송
- Resend 등 외부 이메일 서비스 연동
