# PRD: 관리자 페이지 (Thymeleaf)
- 작성일: 2026-04-05
- 상태: Draft

## 배경 & 목적

사용자가 제보한 장소/이벤트를 관리자가 검수해서 실제 DB에 등록하는 내부 도구가 필요하다.
현재는 구글시트로 데이터를 관리하고 있어 DB 적재 방법도 필요하다.
로그인(B-07)은 추후 붙이고, 기능 먼저 구현한다.

## 요구사항

### 기능 요구사항

#### 대시보드
- [ ] 총 장소 수, 총 이벤트 수 표시
- [ ] 대기 중인 제보 수 (장소/이벤트 각각) 표시

#### 제보 검수
- [ ] 장소 제보 목록 조회 (PENDING / APPROVED / REJECTED 탭 구분)
- [ ] 이벤트 제보 목록 조회 (PENDING / APPROVED / REJECTED 탭 구분)
- [ ] 제보 상세: 사용자가 입력한 원본 내용을 왼쪽에 표시
- [ ] 제보 상세: 오른쪽에 실제 DB 저장용 입력 폼 (제보 내용 자동 채움)
- [ ] 승인: 오른쪽 폼 내용으로 Place 또는 Event 생성 + 제보 상태 APPROVED 변경
- [ ] 거절: 제보 상태 REJECTED 변경

#### 장소/이벤트 CRUD, Excel Import
- 이번 범위 외 → 별도 백로그로 분리 예정

### 비기능 요구사항
- Thymeleaf SSR (별도 프론트 빌드 없음)
- 프론트 디자인 시스템의 색상 팔레트 참고 (어스톤 계열)
- 관리자 사용성 중심 UI (미관보다 효율)
- 인증 없음 (B-07에서 추가 예정) — /admin/** 임시 permitAll

## 가정 & 미확정 사항
- 가정: 카테고리/태그/브랜드는 이미 DB에 시드 데이터로 존재
- 가정: Excel 컬럼 구조는 확정된 구글시트 양식 기준
- 미확정: 장소 삭제 시 소프트 삭제 여부 (→ 일단 하드 삭제로 구현)
- 미확정: 제보 거절 시 사유 입력 필요 여부 (→ 일단 사유 없이 거절)
