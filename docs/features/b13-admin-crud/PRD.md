# PRD: 관리자 장소/이벤트 CRUD 및 만료 관리

- 작성일: 2026-04-14
- 상태: Approved

## 배경 & 목적

현재 관리자 페이지(B-06)는 제보 검수 기능만 있어, 제보를 거치지 않고 관리자가 직접 장소/이벤트를 등록/수정/삭제할 방법이 없다.
초기 데이터 마이그레이션과 운영 편의를 위해 **다건 동시 입력 + 저장 전 프리뷰** 형태의 CRUD를 제공한다.
또한 급하게 콘텐츠를 내려야 할 경우를 위해 **활성/비활성 토글**을 제공하며, 이벤트의 경우 종료일 기반 상태(진행중/만료 임박/만료)를 시각적으로 구분한다.

## 요구사항

### Entity 변경

- [ ] `Place`: `var active: Boolean = true` 컬럼 추가 (default true)
- [ ] `Event`: 기존 `val active` → `var active`로 변경 (이미 컬럼 존재, 토글 가능하도록)
- [ ] **`EventScheduler` 삭제** — 기존에는 스케줄러가 매일 02시 만료 이벤트를 `active=false`로 토글했음. 이제 `active`는 "관리자 수동 공개 여부"로 의미 재정의하고, 만료 여부는 쿼리 시점에 `end_date` 비교로 계산
- [ ] **데이터 마이그레이션**: 기존 `events.active=false` 레코드 중 수동 내림과 만료 자동 전환이 섞여있을 수 있음. 전부 `active=true`로 복원 후 쿼리 필터에 맡기는 것을 권장

### 기능 요구사항

#### 장소 관리 (`/admin/places`)
- [ ] 목록 조회 — 검색(장소명 + 주소 + 카테고리) + 페이징
- [ ] 목록에 활성/비활성 배지 표시
- [ ] 등록 — 다건 입력 (행 추가), 저장 전 프리뷰 모달로 확인 후 최종 저장
- [ ] 수정 — 단건, 프리뷰 모달 동일 적용
- [ ] 활성/비활성 토글
- [ ] 삭제 — 하드 삭제, 확인 다이얼로그

#### 이벤트 관리 (`/admin/events`)
- [ ] 목록 조회 — 페이징, 상태 필터 (전체 / 진행중 / 만료 임박 / 만료)
- [ ] 목록에 활성/비활성 + 진행중/임박/만료 배지 표시
- [ ] 등록 — 다건 입력 + 프리뷰 모달
- [ ] 수정 — 단건 + 프리뷰 모달
- [ ] 활성/비활성 토글
- [ ] 삭제 — 하드 삭제, 확인 다이얼로그

#### 공개 API 필터링 변경
- [ ] 장소 조회: `active = true AND status = OPEN`
- [ ] 장소 상세의 연관 이벤트: `active = true AND (end_date IS NULL OR end_date >= today)`
- [ ] 이벤트 단독 조회: `active = true AND (end_date IS NULL OR end_date >= today)`

#### 대시보드
- [ ] "만료 임박 이벤트 수" 위젯 추가 (종료일 D-7 이내, active=true)

## 가정 & 결정 사항

- **만료 임박 기준**: D-7 이내 (상수화, 추후 조정)
- **만료 이벤트 처리**: 삭제하지 않고 이력으로 남김 (관리자 목록에서만 조회 가능, 공개 API 미노출)
- **Place active vs PlaceStatus 분리**: `active`는 공개 여부(급한 내림), `PlaceStatus`는 실제 운영 상태(OPEN/CLOSED/RELOCATED). 둘 다 AND 조건으로 노출 판단
- **Event endDate null 허용**: 기존 스키마 nullable=true 유지, 공개 조회 시 null이면 상시 진행으로 간주
- **`active` 의미 재정의**: 계산 가능한 값(만료 여부)을 컬럼으로 저장하지 않는다는 원칙. `active`는 수동 공개 플래그, 만료는 `end_date` 비교로 쿼리 시 계산
