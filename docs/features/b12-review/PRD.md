# PRD: 리뷰 기능

- 작성일: 2026-04-14
- 상태: In Progress

## 배경 & 목적

장소/이벤트에 대한 사용자 리뷰를 수집해 콘텐츠 신뢰도를 높인다.
비로그인 방식으로 진입 장벽을 낮추되, 비밀번호 기반 삭제로 최소한의 본인 확인을 제공한다.
닉네임은 프론트엔드에서 랜덤 생성 후 세션/로컬스토리지에 저장하여 관리한다 (서버는 전달받은 닉네임을 그대로 저장).

## 요구사항

### 기능 요구사항

- [ ] 장소/이벤트에 리뷰 작성 (닉네임, 이메일, 비밀번호, 내용)
- [ ] 장소/이벤트별 리뷰 목록 조회 (최신순)
- [ ] 리뷰 삭제 — 작성 시 입력한 비밀번호 검증
- [ ] 관리자 리뷰 관리 — 전체 리뷰 목록 조회 + 비밀번호 없이 삭제

### 비기능 요구사항

- 비밀번호는 BCrypt 해시 저장
- 리뷰 작성 시 대상(Place/Event) 존재 여부 검증
- `(target_type, target_id)` 복합 인덱스

## 가정

- 이메일은 정보 수집용으로만 저장 (검증/알림 없음)
- 닉네임 중복 허용 (비로그인이므로)
- 데이터 규모 소량 — 페이징 불필요 (기존 API 패턴과 동일)

## 범위 외

- 리뷰 수정 기능
- 별점/평점
- 이메일 인증/알림
- 페이징
- 대댓글/답글
