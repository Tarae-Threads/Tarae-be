# PRD: B-04 제보 스키마 설계
- 작성일: 2026-04-05
- 완료일: 2026-04-05
- 상태: Done

## 배경 & 목적

사용자가 새 장소/이벤트를 제보하거나 기존 장소를 업데이트 제안할 수 있어야 한다.
관리자 승인 전까지 PENDING 상태로 보관되며, 승인 시 실제 데이터에 반영된다.

## 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| R1 | 장소 제보(PlaceSubmission) 스키마 확정 | Must |
| R2 | 이벤트 제보(EventSubmission) 스키마 확정 | Must |
| R3 | 제보 상태: PENDING / APPROVED / REJECTED | Must |

## 범위 제외

- 제보 API (B-05)
- 관리자 승인/거절 API (B-06)
