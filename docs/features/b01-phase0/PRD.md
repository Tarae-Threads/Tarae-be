# PRD: B-01 Phase 0 셋업
- 작성일: 2026-04-05
- 완료일: 2026-04-05
- 상태: Done

## 배경 & 목적

본격적인 도메인 개발(Place, Event 등) 전 모든 도메인에서 공통으로 사용할 인프라 코드를 먼저 세팅한다.
이 셋업 없이 도메인 코드를 작성하면 Entity 중복 코드, 예외 처리 누락, 개발 중 인증 차단 문제가 발생한다.

## 요구사항

### 기능 요구사항

- [ ] 모든 Entity가 공통으로 상속할 BaseEntity 제공 (created_at, updated_at 자동 관리)
- [ ] `/admin/**` 경로는 인증 필요, 나머지 모든 경로는 인증 없이 접근 가능
- [ ] 예외 발생 시 일관된 JSON 형식으로 응답

### 비기능 요구사항

- 도메인 코드 없이 독립적으로 빌드/테스트 가능해야 함
- 나중에 실제 인증(JWT 등) 추가 시 SecurityConfig만 수정하면 되는 구조

## 가정 & 미확정 사항

- 가정: 일반 사용자 로그인 없음 (공개 API)
- 가정: 관리자 로그인은 추후 Thymeleaf 기반으로 구현
- 미확정: 에러 응답 필드 구조 (code 필드 포함 여부) → 일단 message + status만

## 범위 외

- JWT, OAuth, 실제 로그인 구현
- Swagger/OpenAPI 설정
- 실제 도메인 Entity 구현
