# Claude Code 에이전트 최적화 — 세팅했는데 안 쓰이는 문제

- 날짜: 2026-04-04

## 상황

에이전트 3개(architect, code-reviewer, product-advisor)와 스킬 4개를 세팅했는데, 실제 세션에서 하나도 자동 발동되지 않았다. 스키마 설계 논의를 하면서 architect 에이전트를 안 쓰고, 백로그 추가 시 /backlog 스킬 안 쓰고 파일을 직접 수정했다.

## 핵심 내용

### 1. 자동 트리거는 믿으면 안 된다
description에 트리거 키워드를 넣어도 Claude가 "판단"해서 안 쓸 수 있다. **CLAUDE.md에 "이 상황에서는 반드시 이 에이전트를 써라"고 명시적으로 적어야** 확실하다.

### 2. 에이전트 페르소나가 성능을 좌우한다
"~~를 수행한다" 식의 매뉴얼 → "너는 ~~한 전문가다" 식의 페르소나로 바꾸면 에이전트 출력 품질이 올라간다. 에이전트는 별도 컨텍스트에서 돌아가기 때문에 "이 사람이 누구인지" 심어줘야 한다.

### 3. 에이전트별 모델 분리
- opus: 깊은 추론 (설계, 전략)
- sonnet: 패턴 매칭 (코드 리뷰)
- haiku: 탐색/읽기 전용 (빠르고 저렴)

전부 같은 모델 쓰면 비용 대비 효율이 나쁘다.

### 4. description은 250자에서 잘린다
핵심 용도를 앞에 배치해야 한다.

### 5. 스킬 vs 에이전트 구분
- 스킬: 정해진 워크플로우 (PRD 작성, 백로그 관리)
- 에이전트: 대화형 전문가 (설계 논의, 기획 브레인스토밍)

기획 논의를 스킬로 만들면 단발 출력으로 끝나고, 에이전트로 만들면 여러 턴 깊은 대화가 된다.

## 왜 중요한가

세팅만 잘 하면 끝이 아니다. 세팅을 계속 고치는 피드백 루프가 핵심이다. `/cc-optimize habits`로 주기적으로 점검하는 습관을 만들어야 한다.
